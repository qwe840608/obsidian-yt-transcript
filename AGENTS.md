# Repository Guidelines

## Project Structure & Module Organization
This directory is an installed Obsidian plugin package for `ytranscript`.
- `main.js`: bundled plugin runtime (generated by esbuild; do not hand-edit unless you are patching a hotfix directly).
- `manifest.json`: plugin metadata (`id`, version, compatibility).
- `styles.css`: plugin UI styles loaded by Obsidian.
- `data.json`: local plugin settings/state in this vault.

There is no source tree (`src/`) or test folder in this directory. Treat this repo as a deployment snapshot.

## Build, Test, and Development Commands
No build scripts are defined in this folder (`package.json` is not present).
Use manual validation in Obsidian:
- Reload plugin: disable/enable the plugin in **Settings -> Community plugins**.
- Reload vault UI: `Ctrl/Cmd+R` in Obsidian.
- Verify behavior: open a note, run plugin command(s), and confirm transcript output and highlighting.

If you maintain upstream source elsewhere, build there and copy updated `main.js`, `manifest.json`, and `styles.css` into this folder.

## Coding Style & Naming Conventions
- CSS: 1 tab indentation (match existing file style).
- Class naming: keep `yt-transcript__*` BEM-like pattern for plugin-specific selectors.
- JSON: preserve existing key casing and formatting in `manifest.json`.
- JavaScript: prefer editing TypeScript source upstream; keep this bundled file changes minimal and well-commented in PR notes.

## Testing Guidelines
Testing is manual in this repository.
- Smoke test: plugin loads without console errors.
- Functional test: YouTube URL parsing and transcript fetch succeed.
- UI test: highlighted transcript spans use `.yt-transcript__highlight` styling.

When changing styles or behavior, include reproducible test steps in the PR.

## Commit & Pull Request Guidelines
Recent history is mostly automated vault snapshots (`vault backup: YYYY-MM-DD ...`).
For intentional development commits, use clear, scoped messages, for example:
- `fix: handle invalid YouTube URLs in transcript fetch`
- `style: adjust transcript highlight contrast`

PRs should include:
- what changed and why,
- affected files (`main.js`, `styles.css`, `manifest.json`),
- manual test steps and results,
- screenshots/GIFs for UI changes.

## Security & Configuration Notes
- Do not commit secrets or personal vault data.
- Review `data.json` before committing; it may contain user-specific settings.
- Keep `manifest.json` version in sync with released plugin behavior.
